## 🚀 **Quick Start Guide**

### **📋 Prerequisites**
- Node.js 18+ and npm
- Python 3.12+
- Supabase account
- Anthropic API key

### **⚡ One-Click Railway Deployment**

[![Deploy on Railway](https://railway.app/button.svg)](https://railway.app/template/your-template-id)

1. **Click the Railway button** above
2. **Connect your GitHub** account
3. **Set environment variables** (see deployment guide)
4. **Deploy automatically** in minutes!

### **🛠️ Local Development Setup**

<details>
<summary><strong>🔧 Backend Setup</strong></summary>

```bash
# Clone and setup backend
cd backend
python -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate
pip install -r requirements.txt

# Configure environment
cp .env.example .env
# Edit .env with your API keys

# Start development server
uvicorn enhanced_api:app --reload --port 8000
```

**🌐 Backend runs at:** `http://localhost:8000`

</details>

<details>
<summary><strong>⚛️ Frontend Setup</strong></summary>

```bash
# Setup frontend
cd frontend
npm install

# Configure environment
cp .env.example .env
# Add your backend URL

# Start development server
npm run dev
```

**🌐 Frontend runs at:** `http://localhost:3002`

</details>

### **🔑 Environment Variables**

<details>
<summary><strong>Backend (.env)</strong></summary>

```env
# Supabase Configuration
SUPABASE_URL=your_supabase_project_url
SUPABASE_KEY=your_supabase_anon_key
SUPABASE_SERVICE_ROLE_KEY=your_supabase_service_role_key

# Database
DATABASE_URL=postgresql://postgres:password@host:5432/postgres

# AI Configuration
ANTHROPIC_API_KEY=your_anthropic_api_key

# App Configuration
ENVIRONMENT=development
```

</details>

<details>
<summary><strong>Frontend (.env)</strong></summary>

```env
# API Configuration
VITE_API_URL=http://localhost:8000

# Optional
VITE_STRIPE_PUBLIC_KEY=your_stripe_public_key
```

</details>

## 🔗 **API Documentation**

### **🎯 Core Endpoints**

| Method | Endpoint | Description |
|--------|----------|-------------|
| `POST` | `/api/v1/generate-playbook` | Generate new messaging playbook |
| `GET` | `/api/v1/playbook-status/{id}` | Check generation status |
| `GET` | `/api/v1/playbook/{id}` | Get playbook details |
| `GET` | `/api/v1/user/playbooks` | Get user's playbooks |
| `DELETE` | `/api/v1/playbook/{id}` | Delete playbook |

### **🔐 Authentication**

| Method | Endpoint | Description |
|--------|----------|-------------|
| `POST` | `/api/v1/auth/register` | User registration |
| `POST` | `/api/v1/auth/login` | User login |
| `GET` | `/api/v1/auth/me` | Get current user |

### **📄 Utilities**

| Method | Endpoint | Description |
|--------|----------|-------------|
| `GET` | `/health` | Health check |
| `GET` | `/api/v1/download-playbook/{id}` | Download PDF |
